寫入檔案(要有header) => 直接做3 或 先做2 再做3  或 啟用override 先做2 插入舊資料 再做3

1.檢查路徑目標檔案
  1.1 找不到檔案
  1.2 找到檔案，但是是空的
  1.3 找到檔案，將檔案暫存起來

2.建立header的string => 代表我需要先備妥一包model反射後的字串

3.建立新資料的string => 需要將物件轉乘raw data的字串

3.若找不到檔案、舊檔案是空的或append = false，就寫入header 與 新資料
=> 需要有一個檔案檢查機制(判斷檔案是否存在/檔案是否擁有Header) => 回傳bool

4.舊檔案的第一行和 header 一致，則寫入 新資料

5.舊檔案的第一行和 header 不一致，則將header插入舊資料第一行後再寫入 新資料

=> 取出當前所有資料 然後再放入Header 再放入 舊資料 再放入 新資料 (啟用 override)


當流程發生改變的時候，需要有一個狀況判定(enum)
enumType:
存在Header,
檔案不存在,
舊資料存在
